WCAG 3 Borrador

Introducción

WCAG 3 es actualmente un borrador incompleto. Está destinado a convertirse en un estándar W3C en unos pocos años. El estándar actual, Pautas de accesibilidad al contenido web (WCAG 2), se presenta en la Descripción general de WCAG 2.

Los documentos WCAG 3 explican cómo hacer que la web sea más accesible para las personas con discapacidad. WCAG 3 se aplica al contenido web, las aplicaciones, las herramientas, la publicación y las tecnologías emergentes en la web.

Esta página se actualizará periódicamente a medida que avance el trabajo en el Borrador de las WCAG 3.

Estado: Borrador para revisión

El primer borrador incompleto y sin pulir, el borrador de trabajo de las Pautas de accesibilidad del W3C (WCAG) 3.0, se publicó en diciembre de 2021. Incluía:

  • Estructura propuesta.
  • Borrador del modelo de conformidad.
  • 5 borradores de pautas de ejemplo.

El último borrador de editores incompleto y no aprobado está disponible.

Enfoque preliminar de WCAG 3

Algunos similares, algunos diferentes

Los objetivos para WCAG 3 incluyen:

  • Ser más fácil de entender.
  • Cubrir más necesidades de los usuarios, incluidas más necesidades de las personas con discapacidades cognitivas.
  • Ser flexible para abordar diferentes tipos de contenido web, aplicaciones, herramientas y organizaciones.

WCAG 3 es similar a las versiones anteriores en algunos aspectos. Tiene parecido:

  • Ojetivo de proporcionar orientación sobre cómo hacer que el contenido web y las aplicaciones sean accesibles para las personas con discapacidad.
  • Requisitos fundamentales y específicos de accesibilidad.

WCAG 3 es muy diferente de las versiones anteriores en algunos aspectos. Tiene:

  • Estructura diferente.
  • Modelo de conformidad diferente.
  • Alcance más amplio, más allá del contenido web.

Estructura

Este borrador de WCAG 3 tiene:

  • Pautas
    • Soluciones a problemas de accesibilidad.
    • Más granular que las pautas en WCAG 2.
    • Cada directriz tiene resultados.
  • Resultados
    • Declaraciones comprobables.
    • Similar a los criterios de éxito en WCAG 2, más granular y más centrado en los resultados deseados que en los medios técnicos para lograrlos.

Más granular significa que son más específicos, en lugar de amplios. Eso generalmente los hace más simples. También significa que habrá más resultados de WCAG 3 que criterios de éxito de WCAG 2.

El material de apoyo de las WCAG 3 incluye:

  • Métodos
    • Maneras específicas de la tecnología para lograr un resultado.
    • Pruebas para el resultado.
    • Similar a Técnicas para WCAG 2.
  • Documentos instructivos
    • Explique más sobre cada directriz, por ejemplo, cómo aborda las necesidades de accesibilidad.
    • Información para diseñadores, desarrolladores, personas nuevas en accesibilidad y planificadores de proyectos.
    • Similar a los documentos de Comprensión para WCAG 2.
  • Categorías funcionales de necesidades funcionales
    • Lista categorizada de necesidades de las personas con discapacidad.
