Árbol de decisiones UNE EN-301 549

El objetivo del árbol de decisión es reducir en gran medida el número inicial de 239 preguntas que serían necesarias para conocer qué requisitos se aplican a cada caso concreto.

En el árbol de decisión, la respuesta de cada pregunta permite identificar si se aplican o no grupos de dos o más requisitos. Y una vez contestada cada pregunta, la respuesta dada permite saber cuál es la siguiente pregunta que debe hacerse.

1. ¿Tiene el sistema TIC funcionalidad cerrada?
El sistema no es compatible con productos de apoyo.
1.1. ¿Se necesita utilizar información visual para usar las funciones del sistema TIC que no son compatibles con lectores de pantalla?
El Sistema TIC presenta información visual que el usuario debe utilizar, y esa información visual no es accesible para los lectores de pantalla.
1.1.1 ¿Se proporciona salida sonora para facilitar el acceso no visual?
Se puede proporcionar acceso no visual de varias formas, pero lo más habitual es utilizar salida sonora (sonidos, notas musicales o voz).
1.1.1.1 ¿Se muestra información en pantalla?
1.1.1.1.1. ¿Se proporciona salida de voz para facilitar el acceso no visual?
Normalmente la salida sonora que permite el acceso sin visión es voz, bien grabada o bien sintetizada.
1.2. ¿La funcionalidad del sistema TIC está cerrada al acceso por teclado?
El sistema no permite conectar teclados (o dispositivos equivalentes).
2. ¿Tiene el sistema TIC elementos accionables? (botones, pulsadores, ...)
El sistema tiene controles físicos o lógicos de cualquier tipo (botones, pulsadores, palancas, etc.).
2.1. ¿Tiene el sistema TIC controles de bloqueo o conmutación?
Tiene controles cuyo estado permanece estable, como por ejemplo la tecla de bloqueo de mayúsculas o los botones de dos estados que se utilizan en las interfaces de usuario de configuración.
3. ¿Proporciona el sistema TIC comunicación bidireccional por voz?
El sistema TIC facilita la comunicación oral entre dos personas.
3.1. ¿Proporciona el sistema TIC funcionalidad RTT?
El sistema ofrece funciones de comunicación de texto en tiempo real (RTT, del inglés real-time text).
3.1.1. ¿Tiene el sistema capacidad para enviar y recibir RTT?
3.2 ¿Permite el sistema TIC comunicación bidireccional de vídeo?
El sistema, además de comunicación por voz, también permite comunicación de vídeo, entonces tendrá que cumplir con requisitos de calidad de vídeo que faciliten la comunicación por lengua de signos o la lectura labial.
3.2.1. ¿Proporciona el sistema TIC información sobre quién está hablando?
4. ¿Tiene el sistema TIC capacidades de vídeo? (Reproducir, grabar, ...)
4.1. ¿Reproduce el sistema TIC vídeo con audio sincronizado?
4.1.1. ¿Muestra el sistema TIC subtítulos?
4.2. ¿El sistema TIC transmite, convierte o graba vídeo con audio sincronizado?
5. ¿El sistema TIC es hardware o tiene componentes hardware?
Se trata de un producto hardware o de un servicio TIC que incluye productos hardware.
5.1. ¿El sistema TIC reproduce voz?
El sistema tiene altavoces que reproducen voz (por ejemplo, un teléfono).
5.1.1. ¿Se sujeta normalmente el sistema TIC cerca del oído?
5.2. ¿Se trata de un sistema TIC estacionario?
5.2.1. ¿Ninguna parte del sistema TIC estacionario obstruye el acceso frontal?
5.2.2. ¿Hay una obstrucción que es parte integral del sistema TIC estacionario?
5.2.3. ¿Hay un obstáculo que sea parte integral del sistema TIC estacionario?
5.2.4. ¿El alcance lateral no tiene obstrucción, o tiene una obstrucción que es parte integral del sistema TIC estacionario?
5.2.4.1. ¿Es la obstrucción (que es parte integral del sistema TIC integrado) menor de 255 milímetros?
5.3. ¿Tiene el sistema TIC elementos accionables de forma mecánica?
La parte hardware del producto TIC tiene botones o teclas mecánicas (que se desplazan físicamente al ser pulsados).
6. ¿El sistema TIC es una página web o contiene páginas web?
Las páginas web tienen que cumplir con los requisitos de EN 301 549 que equivalen al nivel AA de conformidad de WCAG 2.1.
7. ¿El sistema TIC es un documento no web o contiene documentos no web?
Los documentos no web tienen que cumplir con los requisitos de EN 301 549 que equivalen (con alguna excepción) al nivel AA de conformidad de WCAG 2.1. Además hay dos recomendaciones específicas para documentos no web.
8. ¿El sistema TIC es software o contiene software?
El sistema TIC incluye software de plataforma (sistema operativo), software que proporciona una interfaz de usuario, software que es una herramienta de autor o software que opera como producto de apoyo.
8.1. ¿Se trata de software que proporciona una interfaz de usuario?
8.1.1. ¿Es software no web o contiene software no web?
8.2. ¿Es software de plataforma?
El sistema TIC incluye software de plataforma (sistemas operativos, software controlador como drivers…).
8.3. ¿Es una herramienta de autor?
El software permite editar y gestionar contenidos entonces se considera que es una herramienta de autor.
9. ¿Incluye el sistema TIC documentación del producto?
El sistema incluye documentación. Debe cubrir las características de accesibilidad y deberá ser accesible.
10. ¿Se incluyen servicios de apoyo?
El sistema TIC incluye servicios de apoyo como pueden ser los servicios de atención al cliente.
11. ¿El sistema TIC proporciona servicios de intermediación?
El sistema TIC está diseñado para proporcionar servicios de intermediación, deberá cumplir con una serie de requisitos sobre la calidad de esos servicios.
