Árbol de decisiones UNE EN-301 549

El objetivo del árbol de decisión es reducir en gran medida el número inicial de 239 preguntas que serían necesarias para conocer qué requisitos se aplican a cada caso concreto.

En el árbol de decisión, la respuesta de cada pregunta permite identificar si se aplican o no grupos de dos o más requisitos. Y una vez contestada cada pregunta, la respuesta dada permite saber cuál es la siguiente pregunta que debe hacerse.

Q.1. ¿Tiene el sistema TIC funcionalidad cerrada?
Si el sistema no es compatible con productos de apoyo.
Q.1.1. ¿Se necesita utilizar información visual para usar las funciones del sistema TIC que no son compatibles con lectores de pantalla?
El Sistema TIC presenta información visual que el usuario debe utilizar, y esa información visual no es accesible para los lectores de pantalla (que usan las personas ciegas para acceder a información visual del sistema). Si la respuesta a Q1.1 es negativa, entonces se salta a la pregunta Q1.2
Q.1.1.1 ¿Se proporciona salida sonora para facilitar el acceso no visual?
Se puede proporcionar acceso no visual de varias formas, pero lo más habitual es utilizar salida sonora (sonidos, notas musicales o voz). En ese caso se aplican los requisitos siguientes y se plantea la pregunta Q1.1.1.1. Si la respuesta a Q1.1.1 es negativa, se salta a la pregunta Q1.2.
Q.1.1.1.1 ¿Se muestra información en pantalla?
Q.1.1.1.1.1. ¿Se proporciona salida de voz para facilitar el acceso no visual?
Normalmente la salida sonora que permite el acceso sin visión es voz, bien grabada o bien sintetizada. En caso de que se use voz, se aplican los requisitos siguientes. Y la siguiente pregunta será Q1.2, sea cual sea la respuesta a la pregunta Q1.1.1.1.
Q.1.2. ¿La funcionalidad del sistema TIC está cerrada al acceso por teclado?
Si el sistema no permite conectar teclados (o dispositivos equivalentes), entonces se aplican dos requisitos más de EN 301 549. Y la siguiente pregunta será Q2, sea cual sea la respuesta a Q1.2.
Q.2. ¿Tiene el sistema TIC elementos accionables? (botones, pulsadores, ...)
Esto será así si el sistema tiene controles físicos o lógicos de cualquier tipo (botones, pulsadores, palancas, etc.). Si es así se aplican los requisitos siguientes. La siguiente pregunta será la pregunta Q3 en cualquier caso.
Q.2.1. ¿Tiene el sistema TIC controles de bloqueo o conmutación?
La pregunta se refiere a controles cuyo estado permanece estable, como por ejemplo la tecla de bloqueo de mayúsculas o los botones de dos estados que se utilizan en las interfaces de usuario de configuración. Si es así, entonces se aplican dos requisitos más. La siguiente pregunta será la pregunta Q4 en cualquier caso.
Q.3. ¿Proporciona el sistema TIC comunicación bidireccional por voz?
La pregunta se refiere a si el sistema TIC facilita la comunicación oral entre dos personas. En caso afirmativo se aplican varios requisitos de los capítulos 6 y 13 de EN 301 549 y la siguiente pregunta será la Q4.1. En caso negativo se pasa a la pregunta Q5.
Q.3.1. ¿Proporciona el sistema TIC funcionalidad RTT?
Si el sistema ofrece funciones de comunicación de texto en tiempo real (RTT, del inglés real-time text) entonces tiene que cumplir con unos criterios de calidad sobre este tipo de comunicación. Si la respuesta a Q4.1 es afirmativa la siguiente pregunta es Q4.1.1. En caso contrario la siguiente pregunta es Q4.2.
Q.3.1.1. ¿Tiene el sistema capacidad para enviar y recibir RTT?
Q.3.2 ¿Permite el sistema TIC comunicación bidireccional de vídeo?
Si el sistema, además de comunicación por voz, también permite comunicación de vídeo, entonces tendrá que cumplir con requisitos de calidad de vídeo que faciliten la comunicación por lengua de signos o la lectura labial. Sea cual sea la respuesta a esta pregunta, la siguiente pregunta será Q5
Q.3.2.1. ¿Proporciona el sistema TIC información sobre quién está hablando?
[Q14] ¿Tiene el sistema TIC capacidades de vídeo? (Reproducir, grabar, ...)
Si la respuesta es afirmativa se aplican los requisitos siguientes. La siguiente pregunta será Q6 en cualquier caso.
[Q15] ¿Reproduce el sistema TIC vídeo con audio sincronizado?
[Q16] ¿Muestra el sistema TIC subtítulos?
[Q17] ¿El sistema TIC transmite, convierte o graba vídeo con audio sincronizado?
[Q18] ¿El sistema TIC es hardware o tiene componentes hardware?
Si se trata de un producto hardware o de un servicio TIC que incluye productos hardware, entonces se aplican algunos requisitos generales de software y se plantea la pregunta Q6.1. En caso contrario se salta a la pregunta Q7.
[Q19] ¿El sistema TIC reproduce voz?
Si el sistema tiene altavoces que reproducen voz (por ejemplo, un teléfono), entonces se aplican requisitos relacionados con la calidad del sonido. La siguiente pregunta será Q6.2 en cualquier caso.
[Q20] ¿Se sujeta normalmente el sistema TIC cerca del oído?
[Q21] ¿Se trata de un sistema TIC estacionario?
[Q22] ¿Ninguna parte del sistema TIC estacionario obstruye el acceso frontal?
[Q23] ¿Hay una obstrucción que es parte integral del sistema TIC estacionario?
[Q24] ¿Hay un obstáculo que sea parte integral del sistema TIC estacionario?
[Q25] ¿El alcance lateral no tiene obstrucción, o tiene una obstrucción que es parte integral del sistema TIC estacionario?
[Q26] ¿Es la obstrucción (que es parte integral del sistema TIC integrado) menor de 255 milímetros?
[Q27] ¿Tiene el sistema TIC elementos accionables de forma mecánica?
Si la parte hardware del producto TIC tiene botones o teclas mecánicas (que se desplazan físicamente al ser pulsados), entonces se aplican requisitos sobre esos botones o teclas. La siguiente pregunta será Q7 en cualquier caso.
[Q28] ¿El sistema TIC es una página web o contiene páginas web?
Las páginas web tienen que cumplir con los requisitos de EN 301 549 que equivalen al nivel AA de conformidad de WCAG 2. La siguiente pregunta sería Q8 en cualquier caso.
[Q29] ¿El sistema TIC es un documento no web o contiene documentos no web?
Los documentos no web tienen que cumplir con los requisitos de EN 301 549 que equivalen (con alguna excepción) al nivel AA de conformidad de WCAG 2. Además hay dos recomendaciones específicas para documentos no web. La siguiente pregunta sería Q9 en cualquier caso.
[Q30] ¿El sistema TIC es software o contiene software?
Si el sistema TIC incluye software de plataforma (sistema operativo), software que proporciona una interfaz de usuario, software que es una herramienta de autor o software que opera como producto de apoyo se aplican unos requisitos y la pregunta siguiente será Q9.1. En otro caso la pregunta siguiente será Q10.
[Q31] ¿Se trata de software que proporciona una interfaz de usuario?
[Q32] ¿Es software no web o contiene software no web?
[Q33] ¿Es software de plataforma?
Si el sistema TIC incluye software de plataforma (sistemas operativos, software controlador como drivers…) entonces tiene que cumplir tres requisitos específicos. La siguiente pregunta será Q9.3 en cualquier caso.
[Q34] ¿Es una herramienta de autor?
Si el software permite editar y gestionar contenidos entonces se considera que es una herramienta de autor y se aplican requisitos específicos de accesibilidad. La siguiente pregunta será Q10 en cualquier caso.
[Q35] ¿Incluye el sistema TIC documentación del producto?
Si el sistema incluye documentación, ésta tendrá que cubrir las características de accesibilidad y deberá ser accesible. La siguiente pregunta será Q11 en cualquier caso.
[Q36] ¿Se incluyen servicios de apoyo?
Si el sistema TIC incluye servicios de apoyo como pueden ser los servicios de atención al cliente, entonces esos servicios tienen que cumplir tres requisitos de accesibilidad. La siguiente pregunta será Q12 en cualquier caso.
[Q37] ¿El sistema TIC proporciona servicios de intermediación?
Si el sistema TIC está diseñado para proporcionar servicios de intermediación, deberá cumplir con una serie de requisitos sobre la calidad de esos servicios. Ésta es la última pregunta del árbol de decisión.
